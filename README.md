## Детекция мошеннических транзакций
Данные: Transactions Fraud Datasets (https://www.kaggle.com/datasets/computingvictor/transactions-fraud-datasets) - транзакции по картам: таблицы транзакций, клиентов и карт + метки фрода из JSON.

| Признак | Описание |
|---------|----------|
| `transaction_id` | Уникальный идентификатор транзакции |
| `customer_id` | Идентификатор клиента |
| `card_id` | Идентификатор карты |
| `transaction_date` | Дата и время транзакции |
| `amount` | Сумма транзакции |
| `merchant` | Магазин / получатель платежа |
| `transaction_type` | Тип транзакции (покупка, снятие и т.д.) |
| `location` | Место совершения транзакции |
| `fraud` | Флаг мошенничества (1 / 0) |

**Что сделано:**
- объединение трех таблиц в один датасет
- очистка денежных полей и создание признаков из дат
- кодирование категорий (one-hot + frequency encoding)
- удаление коллинеарных признаков (|corr| > 0.95)
- логарифмирование суммы, обработка отрицательных значений
- разбиение и сохранение выборок train / val / test (70 / 20 / 10)<br>
  для обучения модели прогнозирования мошеннических транзакций

выявлен сильный дисбаланс классов: доля фрода составляет 0.32.

Стек: Python, pandas, numpy, matplotlib/seaborn.
